# Prompt Optimizer · 提示词优化器

将用户的原始提示词（Prompt）优化为结构化、专业化、目标明确的高质量提示词，提升 AI 输出质量。

## 触发条件

- 用户输入以 `/optimize` 为前缀
- 用户明确说"优化这个 prompt"、"帮我写个提示词"、"优化提示词"等
- 用户要求生成/改进用于投喂给 AI 的指令

## 核心流程

1. **解析输入**：提取 `/optimize` 后的内容，检测语言，识别参数
2. **信息评估**：评估信息完整性，不足则发起渐进式追问（最多 1-2 轮，用户可随时跳过）
3. **框架匹配**：用户指定框架则使用，否则自动匹配最佳框架
4. **多版本生成**：默认生成 3 个版本（专业深度版 / 平衡实用版 / 简洁高效版）
5. **对比展示**：三版本并列展示 + 对比总结表 + 优化亮点
6. **用户确认**：选择版本 / 融合版本 / 修改调整 / 切换框架 / 直接执行

## 三大优化框架

| 框架 | 适用场景 | 特点 |
|------|---------|------|
| **CRISPE** | 复杂任务、专业领域 | 六维度：角色-背景-输入-风格-目标-示例 |
| **ROSES** | 文案写作、创意生成 | 五维度：角色-目标-场景-要素-风格 |
| **结构化多层优化** | 通用任务、简单任务 | 七维度：目标-角色-任务-约束-格式-示例-标准 |

> 详细框架说明见 `references/` 目录。

## 支持的参数

| 参数 | 作用 |
|------|------|
| `--crispe` / `--roses` / `--structured` | 指定优化框架 |
| `--quick` / `--fast` | 快速模式：不追问 + 只输出简洁版 |
| `--single` | 单版本输出（默认平衡实用版） |
| `--detail` / `--full` | 仅输出专业深度版 |

## 三版本说明

| 版本 | 定位 | 推荐指数 |
|------|------|---------|
| 🅰️ 专业深度版 | 最详细，质量最高，prompt 较长 | ⭐⭐⭐ |
| 🅱️ 平衡实用版 ⭐推荐 | 均衡适中，大多数场景首选 | ⭐⭐⭐⭐⭐ |
| 🅲 简洁高效版 | 最精简，省 token，适合简单任务 | ⭐⭐⭐ |

## 模板库

### 文案写作类
公众号/自媒体文章、小红书种草文案、广告语/Slogan、短视频脚本、邮件通知

### 创意生成类
故事/小说创作、品牌命名、活动策划、广告创意、诗歌/歌词

> 模板文件位于 `templates/` 目录。

## 交互规范

### 渐进式追问
- 一次问 2-3 个关键问题，附选项引导
- 最多追问 1-2 轮
- 用户说"直接优化"、"先这样"立即停止追问

### 多版本展示
- 三版本按 A→B→C 顺序排列
- B 版标注「⭐推荐」
- 末尾附版本对比总结表和优化亮点

### 用户选择
支持以下操作：
1. 选择 A/B/C 版本直接使用
2. 融合多个版本（如"用A的详细+B的语气"）
3. 提出修改意见，迭代优化
4. 切换框架重新生成
5. 直接用推荐版本执行任务

## 优化原则

1. **目标明确**：优化后的 prompt 必须有清晰的目标
2. **角色具体**：角色设定专业具体，不笼统
3. **约束清晰**：明确能做什么、不能做什么
4. **结构分层**：使用编号、标题、符号分层
5. **保留原意**：优化是增强表达，不改变用户需求
6. **语言一致**：优化后的语言与输入保持一致
7. **版本差异**：多版本输出要有明显的定位差异
