# 使用指南 · Prompt Optimizer

## 快速开始

### 基本用法
在输入框中输入 `/optimize` 加上你想要优化的提示词即可。

```
/optimize 帮我写一篇关于AI的文章
```

系统会自动：
1. 评估你的信息是否充足
2. 如果信息不足，会追问 2-3 个关键问题（也可以直接跳过）
3. 自动匹配最佳优化框架
4. 生成 3 个不同深度的优化版本供你选择

---

## 参数大全

### 框架参数

| 参数 | 对应框架 | 适用场景 |
|------|---------|---------|
| `--crispe` | CRISPE 全面型 | 复杂任务、专业领域 |
| `--roses` | ROSES 创意文案型 | 文案写作、创意生成 |
| `--structured` | 结构化多层优化 | 通用任务、简单任务 |

**示例**：
```
/optimize --crispe 帮我分析这份销售数据并给出改进建议
/optimize --roses 写一个奶茶店的宣传文案
/optimize --structured 帮我整理一份旅行清单
```

### 版本控制参数

| 参数 | 效果 | 适用场景 |
|------|------|---------|
| 默认（无参数） | 输出 3 个版本（A专业深度 + B平衡实用 + C简洁高效） | 大多数情况，想对比选择 |
| `--single` | 只输出 B 版（平衡实用版） | 知道自己要什么，不想看太多选项 |
| `--detail` / `--full` | 只输出 A 版（专业深度版） | 对质量要求极高，复杂任务 |
| `--quick` / `--fast` | 跳过追问 + 只输出 C 版（简洁高效版） | 快速出稿，简单任务，省时间 |

**示例**：
```
/optimize --single 写一篇小红书笔记
/optimize --detail 写一份产品需求文档
/optimize --quick 帮我想几个广告语
```

### 组合使用
参数可以组合使用：

```
/optimize --roses --detail 写一篇品牌故事
（用 ROSES 框架，只输出专业深度版）

/optimize --crispe --quick 整理一份会议纪要
（用 CRISPE 框架 + 快速模式，输出简洁版）
```

```

```

如果你的信息不够充分，系统会问几个小问题：

```
为了让优化效果更好，我需要再确认几个小问题（也可以直接说"先优化"跳过）：

1. 你的咖啡店主要客群是？（比如：大学生 / 白领上班族 / 家庭用户）
2. 想要什么风格的文案？（比如：年轻活泼 / 文艺清新 / 高端质感）
3. 主要用在什么渠道？（比如：朋友圈 / 门店海报 / 外卖平台）
```

**你可以**：
- 回答问题，让优化更精准
- 直接说"先优化"或"直接优化"，跳过追问

| **直接执行** | "直接用B版开始写"、"执行吧" | 用推荐版本直接开始执行任务 |

- 干净的最终版本（方便复制）
- 是否需要直接执行的询问

## 三版本怎么选？

### 🅰️ 专业深度版
**什么时候选？**
- 任务比较复杂，对输出质量要求很高
- 需要 AI 稳定、精准地输出
- 有足够的 token 预算
- 第一次用，想看看最完整的效果

**特点**：
- 角色设定非常详细（经验、背景、案例都有）
- 约束条件很细致（字数、风格、禁忌都列清楚）
- 输出结构完整，每个维度都覆盖
- prompt 较长，但输出质量最稳定

### 🅱️ 平衡实用版 ⭐推荐
**什么时候选？**
- 大多数日常任务都选这个
- 想要质量和效率的平衡
- 不确定选哪个的时候，选它准没错

**特点**：
- 核心维度都有，但不冗余
- 角色设定合理，不会太夸张
- 约束条件恰到好处
- 性价比最高，推荐大多数用户使用

### 🅲 简洁高效版
**什么时候选？**
- 任务比较简单，目标很明确
- 想省 token，追求效率
- 对 prompt 长度敏感
- 快速出稿，先看看效果

**特点**：
- 只保留最核心的要素
- prompt 短小精悍
- 输出速度快，省 token
- 适合简单任务或快速验证

## 使用技巧

### 技巧 1：信息越充分，优化效果越好

虽然系统会追问补充，但如果你能一次性提供更多信息，效果会更好。

**不太好的输入**：
```
/optimize 写文案
```

**更好的输入**：
```
/optimize 写一个奶茶店的开业宣传文案，目标受众是大学生，价格亲民，主打手作鲜果茶，用在朋友圈和门店海报
```

### 技巧 2：想快速出稿用 --quick

如果你的需求很简单，不想被追问，直接加 `--quick` 参数：

```
/optimize --quick 帮我想几个奶茶店名字
```

### 技巧 3：不确定用哪个框架？让系统自动选

不用纠结用哪个框架，系统会根据你的内容自动匹配。如果对结果不满意，再切换框架也不迟。

### 技巧 4：多版本可以融合使用

三个版本各有优势，你可以自由组合：

```
"用 A 版的详细程度，但语气要像 C 版一样简洁"
"B 版不错，但角色设定参考 A 版再加强一点"
```

### 技巧 5：优化后直接执行

确认满意后，可以直接让 AI 开始执行，不用复制粘贴：

```
"就用 B 版，直接开始写吧"
```

## 渐进式追问说明

### 为什么要追问？
信息越充分，优化出来的 prompt 越精准，AI 输出的质量就越高。适当补充几个关键信息，可以大幅提升最终效果。

### 会问多少问题？
- 通常一次问 **2-3 个**最关键的问题
- 最多追问 **1-2 轮**，不会一直问下去
- 你随时可以说"直接优化"跳过追问

### 都问些什么？
根据不同场景问不同的问题：

| 场景 | 常见问题 |
|------|---------|
| 文案类 | 目标受众？风格偏好？使用渠道？字数要求？核心卖点？ |
| 创意类 | 主题方向？风格调性？数量要求？预算/限制？参考案例？ |
| 通用类 | 输出格式？专业领域？约束条件？时间要求？ |

### 不想回答怎么办？
直接说以下任意一句即可跳过：
- "直接优化"
- "先优化吧"
- "不用问了"
- "按默认来"

## 常见问题

### Q1：为什么优化后的 prompt 比原来长这么多？
**A**：这是正常的。优化的核心就是补充角色、约束、格式等信息，让 AI 更准确地理解你的需求。更长的 prompt 往往能带来更好的输出质量。
> 如果你想要简洁版，可以选择 **C 版简洁高效版**，或者用 `--quick` 参数。

### Q2：三个版本有什么区别？我该选哪个？
**A**：简单来说：
- **A 版**：最详细，质量最高，但 prompt 最长
- **B 版**：平衡之选，大多数场景都适用（推荐）
- **C 版**：最简洁，省 token，适合简单任务
> 不确定选哪个就选 B 版，不满意再换。

### Q3：可以自己组合不同版本的特点吗？
**A**：当然可以！你可以说"用A的结构+B的语气"、"C版再加点内容"，系统会根据你的要求融合生成新版本。

### Q4：自动识别的框架不对怎么办？
**A**：你可以手动指定框架，在输入时加上 `--crispe`、`--roses` 或 `--structured` 参数即可。

### Q5：追问太多了，可以关掉吗？
**A**：可以。使用 `--quick` 或 `--fast` 参数，系统就不会追问，直接基于已有信息快速优化。
> 或者在追问时直接说"直接优化"也可以跳过。

### Q6：模板库可以自定义吗？
**A**：当前版本暂不支持用户自定义模板，但你可以通过修改优化结果来满足个性化需求。未来版本会考虑增加自定义模板功能。

### Q7：支持英文 prompt 优化吗？
**A**：支持。系统会自动检测输入语言，英文输入会用英文优化，保持语言一致性。

### Q8：优化结果可以保存吗？
**A**：当然可以！你可以把满意的优化结果复制保存下来，以后遇到类似任务直接使用或稍作修改即可。
