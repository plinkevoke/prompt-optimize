# 多版本输出机制

## 设计理念
不同用户对 prompt 的需求不同：
- 有的追求极致质量，不在乎 prompt 长度
- 有的追求效率，希望 prompt 简洁
- 大多数用户需要一个平衡点

因此，一次提供 3 个不同深度的版本，让用户选择最适合自己的。

## 三版本策略

| 版本 | 定位 | 详细程度 | 特点 | 适用人群 |
|------|------|---------|------|---------|
| **A · 专业深度版** | 详细全面 | ⭐⭐⭐⭐⭐ | 维度完整、约束细致、角色深入，输出质量最稳定 | 追求极致质量、复杂任务 |
| **B · 平衡实用版** | 均衡适中 | ⭐⭐⭐ | 核心维度齐全，兼顾质量和简洁 | 大多数用户（默认推荐） |
| **C · 简洁高效版** | 精简快速 | ⭐ | 只保留最关键要素，短小精悍 | 简单任务、追求效率 |

## 版本差异化原则

1. **核心目标一致**：三个版本的任务目标必须完全一致，不能偏离用户原意
2. **深度梯度明显**：详细程度从 A 到 C 逐级递减，差异要明显
3. **框架统一**：三个版本使用同一个框架，只是深度不同
4. **各有侧重**：
   - A 版：重细节、重约束、重角色深度
   - B 版：重平衡、重实用、重性价比
   - C 版：重简洁、重效率、重核心信息
5. **避免换汤不换药**：不能只是删减几句话，要有结构上的差异

## 各版本差异对照（以 ROSES 框架为例）

| 维度 | A 专业深度版 | B 平衡实用版 | C 简洁高效版 |
|------|------------|------------|------------|
| 角色 | 详细：经验+背景+代表作品+方法论 | 适中：身份+擅长领域 | 精简：一句话身份 |
| 目标 | 详细：多个目标+量化指标 | 适中：主要目标+效果描述 | 精简：一句话目标 |
| 场景受众 | 详细：场景+画像+心理+触媒 | 适中：场景+受众+特点 | 精简：受众+场景 |
| 要素约束 | 详细：卖点+风格+字数+禁忌+... | 适中：核心要素+主要约束 | 精简：关键要求 |
| 风格结构 | 详细：风格+语气+结构+示例 | 适中：风格+结构要求 | 精简：风格+输出格式 |

## 版本控制参数

| 参数 | 效果 | 适用场景 |
|------|------|---------|
| 默认（无参数） | 输出 3 个版本 | 大多数情况，想对比选择 |
| `--single` | 只输出 B 版（平衡实用版） | 知道自己要什么，不想看太多选项 |
| `--detail` / `--full` | 只输出 A 版（专业深度版） | 对质量要求极高，复杂任务 |
| `--quick` / `--fast` | 跳过追问 + 只输出 C 版 | 快速出稿，简单任务 |

## 版本融合功能

用户可以要求融合多个版本的特点，例如：
- "用 A 的详细程度 + C 的简洁语气"
- "用 B 的结构 + A 的角色设定"
- "C 版太简了，加一点 B 的内容"

系统根据用户的融合要求，生成新的定制版本。

## 版本对比总结表

每次输出时附带版本对比表，帮助用户快速选择：

```
| 版本 | 详细程度 | 适用场景 | 推荐指数 |
|------|---------|---------|---------|
| A 专业深度版 | ⭐⭐⭐⭐⭐ | 复杂任务、高质量要求 | ⭐⭐⭐ |
| B 平衡实用版 | ⭐⭐⭐ | 大多数场景 | ⭐⭐⭐⭐⭐ |
| C 简洁高效版 | ⭐ | 简单任务、省token | ⭐⭐⭐ |
```
