# 渐进式追问机制

## 设计理念
信息越充分，优化效果越好。但过多的追问会让用户烦躁，因此需要平衡：
- 信息不足时，适度追问补充关键信息
- 控制追问轮次，最多 1-2 轮
- 始终提供跳过选项

## 信息完整性评估

### 评估维度

| 场景类型 | 必备信息（缺一则需追问） | 加分信息（有则更好） |
|---------|----------------------|-------------------|
| **文案类** | ① 文案类型/品类 ② 产品/品牌名称 | 目标受众、使用场景、字数要求、风格偏好、核心卖点、竞品信息 |
| **创意类** | ① 创意类型 ② 主题/方向 | 目标受众、风格偏好、数量要求、预算限制、参考案例 |
| **通用类** | ① 明确的任务目标 | 输出格式、时间要求、专业领域、约束条件 |

### 评估规则
- 缺少必备信息 → 必须追问
- 缺少 3 项以上加分信息 → 建议追问
- 加分信息覆盖率 ≥ 60% → 信息充足，直接优化

## 追问设计原则

1. **少而精**：一次只问 2-3 个最关键的问题
2. **有引导**：问题要具体，提供选项/示例降低回答门槛
3. **控轮次**：最多追问 1-2 轮，避免用户烦躁
4. **可跳过**：始终提供"直接优化"的跳过选项
5. **讲价值**：说明补充信息能带来什么好处

## 追问话术模板

```
为了让优化效果更好，我需要再确认几个小问题（也可以直接说"先优化"跳过）：

1. <问题1>？（比如：选项A / 选项B / 选项C）
2. <问题2>？（比如：选项A / 选项B / 选项C）
3. <问题3>？
```

## 各场景常见追问问题

### 文案类
1. 目标受众主要是哪些人？
2. 有没有偏好的风格调性？
3. 主要用在什么渠道/场景？
4. 字数或篇幅有要求吗？
5. 核心卖点/优势是什么？

### 创意类
1. 主题方向有没有更具体的想法？
2. 想要什么风格调性？
3. 数量上有要求吗？
4. 有没有参考案例或喜欢的风格？
5. 有没有预算或限制条件？

### 通用类
1. 输出格式有要求吗？（比如表格/列表/段落）
2. 有没有时间或篇幅限制？
3. 涉及什么专业领域吗？
4. 有没有特别需要注意的约束条件？

## 跳过追问的触发条件

- 用户使用 `--quick` 或 `--fast` 参数
- 用户明确说"不用问了"、"直接优化"、"先这样"
- 用户提供的信息已经非常充分（加分信息覆盖率 ≥ 60%）
- 用户连续两次模糊回答 → 停止追问，基于已有信息优化
